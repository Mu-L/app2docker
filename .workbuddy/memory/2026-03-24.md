## Portainer API 接口对比分析

### 任务背景
用户要求对比最新的Portainer API文档与当前页面和后台接口实现,找出不匹配的问题。

### 完成工作

#### 1. 文档分析
- 阅读了 `docs/portainer-api.md` (Portainer 2.33.3 API文档)
- 分析了后台实现: `backend/portainer_client.py`, `backend/routes.py`, `backend/agent_host_manager.py`
- 检查了前端实现: `frontend/src/components/DeployTaskManager.vue`, `frontend/src/components/HostManager.vue`

#### 2. 发现的问题

**严重问题 (P0)**:
1. **创建容器API不存在** - 后台使用 `POST /docker/containers/create`,但文档明确说明此接口在Portainer 2.33.3中返回404,不支持直接创建容器
2. **缺少Stack类型参数** - 创建Stack时未指定 `type=2` 参数,可能导致创建失败

**重要问题 (P1)**:
1. **更新Stack参数不完整** - 缺少 `Prune: false` 参数
2. **Stack启动/停止接口未验证** - 文档标注待验证,需要测试

**优化建议 (P2)**:
1. 未使用Stack状态检测 (Status字段: 1=运行中, 2=已停止)
2. 前端可增加Portainer部署模式提示

#### 3. 核心发现
- Portainer 2.33.3 **不支持直接创建容器**,必须通过Stack API创建
- 认证方式: API Key 和 JWT Token 都支持,当前使用API Key是正确的
- Stack创建必须包含 `type=2` 参数 (Docker Compose类型)

#### 4. 提供的解决方案
- 详细的代码修复方案,包括:
  - 将 `deploy_container` 方法改为使用Stack API
  - 添加 `type=2` 参数
  - 优化Stack更新参数
- 测试建议和验证步骤
- 优先级行动清单
